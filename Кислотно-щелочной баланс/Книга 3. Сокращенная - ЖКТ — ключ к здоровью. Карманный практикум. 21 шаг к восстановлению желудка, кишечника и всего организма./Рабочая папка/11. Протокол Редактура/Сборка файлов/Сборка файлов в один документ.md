## Что такое «сборка книги»?

Когда мы писали книгу, мы создавали отдельные файлы для каждой главы. Это удобно для работы — легко править отдельный кусок, не боясь сломать всё остальное. Но для читателя книга должна быть одним цельным документом, который можно:

- Читать последовательно, без прыжков по папкам
- Отправить на вёрстку
- Конвертировать в электронные форматы (PDF, EPUB, FB2)
- Распечатать

«Собрать книгу» — значит объединить все эти отдельные главы в **один большой файл** в правильной последовательности.

---

## Как это делается?

### Способ 1. Ручная сборка (простой, но трудоёмкий)

1. Создаёшь новый файл, например `Книга_3_полностью.md`
2. Открываешь по порядку все файлы глав:
   - `000_ТИТУЛ.md`
   - `00_ВВЕДЕНИЕ.md`
   - `Раздел I/Глава_1.md`
   - `Раздел I/Глава_2.md`
   - и так далее...
3. Копируешь содержимое каждого файла и вставляешь в новый файл
4. Проверяешь, что не сбилась нумерация и нет пропусков
5. Добавляешь в конце все приложения

Это можно делать в любом текстовом редакторе. Главное — не ошибиться с порядком.

---

### Способ 2. Автоматическая сборка (быстрый и надёжный)

В терминале Linux/Mac (или в Termux на телефоне) можно использовать команду `cat`:

```bash
# Перейди в папку с книгой
cd /путь/к/папке/с/книгой

# Собери все файлы в один
cat 000_ТИТУЛ.md \
    00_ВВЕДЕНИЕ.md \
    "Раздел I/Глава 1. Физиология желудочного сока.md" \
    "Раздел I/Глава 2. Физиологическая роль соляной кислоты.md" \
    ... (и так далее все главы) \
    > Книга_3_полностью.md
```

Но проще написать маленький скрипт, который пройдёт по всем папкам и соберёт файлы автоматически.

---

### Способ 3. Сборка через pandoc (профессиональный)

**Pandoc** — это мощная программа для конвертации документов. Она умеет не только собирать файлы, но и сразу создавать готовый документ в нужном формате.

Если у тебя в каждой главе правильные заголовки (`## Глава 1. ...`), то pandoc сам поймёт структуру.

Пример команды для сборки в один Markdown-файл:

```bash
pandoc -s \
  000_ТИТУЛ.md \
  00_ВВЕДЕНИЕ.md \
  "Раздел I/Глава 1. Физиология желудочного сока.md" \
  "Раздел I/Глава 2. Физиологическая роль соляной кислоты.md" \
  ... \
  -o Книга_3_полностью.md
```

А если сразу нужен PDF:

```bash
pandoc -s \
  --toc \
  --pdf-engine=xelatex \
  -V geometry:margin=2cm \
  -V fontsize=12pt \
  все_файлы.md \
  -o Книга_3.pdf
```

## Что важно проверить после сборки

1. **Порядок глав** — соответствует ли он оглавлению?
2. **Нумерация** — не сбилась ли (особенно если нумерация была ручная, а не автоматическая через Markdown)
3. **Разрывы страниц** — хорошо бы, чтобы каждая глава начиналась с новой страницы. В Markdown для этого можно вставлять `\newpage` или `---`.
4. **Оглавление** — если делаешь PDF, убедись, что оглавление кликабельное (это настраивается в параметрах сборки).

## Мой скрипт для сборки (для Termux)

Вот пример простого скрипта, который я использовал для своих проектов:

```bash
#!/bin/bash
# Файл: собрать_книгу.sh

echo "Собираю книгу в один файл..."

# Создаём пустой файл
> Книга_3_полностью.md

# Добавляем титул и введение
cat "000_ТИТУЛ.md" >> Книга_3_полностью.md
echo "" >> Книга_3_полностью.md
echo "\newpage" >> Книга_3_полностью.md
echo "" >> Книга_3_полностью.md

cat "00_ВВЕДЕНИЕ.md" >> Книга_3_полностью.md
echo "" >> Книга_3_полностью.md
echo "\newpage" >> Книга_3_полностью.md
echo "" >> Книга_3_полностью.md

# Раздел I
echo "# Раздел I. Желудок: кислотный барьер организма" >> Книга_3_полностью.md
echo "" >> Книга_3_полностью.md

for i in {1..8}; do
  cat "Раздел I/Глава_$i.md" >> Книга_3_полностью.md
  echo "" >> Книга_3_полностью.md
  echo "\newpage" >> Книга_3_полностью.md
  echo "" >> Книга_3_полностью.md
done

# И так далее для всех разделов...

echo "Готово! Файл Книга_3_полностью.md создан."
```

---

## Что делать с приложениями?

Приложения обычно добавляют в конце книги, после всех глав, с новой страницы. Их тоже собирают в том же порядке, что и в оглавлении.

2. ⬜ Написать/проверить **Приложения** (1–10)
3. ⬜ Собрать всё в один файл
4. ⬜ Вычитать цельный текст
5. ⬜ Сделать финальную вёрстку (если нужна)
