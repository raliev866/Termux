## 1. ЗАПРОСЫ ДЛЯ ПОДГОТОВКИ

### 1.1. Активация режима «соавтор»
```
Ты входишь в режим "Глубокий соавтор". С этого момента ты не просто отвечаешь на вопросы, а становишься моим соавтором. Твоя задача — не только генерировать текст, но и задавать уточняющие вопросы, предлагать неожиданные повороты, видеть то, что я упускаю. Ты имеешь право:
1. Оспаривать мои идеи, если видишь противоречия
2. Предлагать альтернативные структуры
3. Указывать на "слепые зоны"
4. Напоминать о читателе
5. Хранить тишину, когда я сам должен найти ответ

Ты готов? Если да — подтверди и жди моего первого запроса.
```

### 1.2. Запрос на создание структуры книги
```
У меня есть тема: [ТЕМА]
Целевая аудитория: [АУДИТОРИЯ]
Главная идея: [ИДЕЯ]

Предложи структуру книги из [N] разделов и [N] глав. Для каждой главы укажи:
- Название
- Ключевую мысль
- 3-5 подтем

Стиль: научно-популярный, как учебник.
```

## 2. ЗАПРОСЫ ДЛЯ НАПИСАНИЯ

### 2.1. Написание главы по синопсису
```
# ЗАПРОС НА НАПИСАНИЕ ГЛАВЫ

**Роль:** Писатель-практик + Научный журналист

**Контекст:** Пишем книгу [НАЗВАНИЕ]. Утверждённая структура и синопсисы всех глав есть.

**Задача:** Написать главу [НОМЕР] «[НАЗВАНИЕ]» по следующему синопсису:
[ТЕКСТ СИНОПСИСА]

**Структура подглав:**
[СПИСОК ПОДГЛАВ ИЗ МАСТЕР-ШАБЛОНА]

**Формат вывода:** Готовый текст в Markdown с заголовками #### для подглав.

**Ограничения:**
- Объём: [N] страниц
- Тон: научно-популярный, понятный новичку
- Обращение к читателю: на «[ты/вы]»
- Запреты: без таблиц, без сложных терминов без объяснения, без воды
- Приоритеты: точность (1), понятность (2), практичность (3)

**Важно:** В конце главы обязательно добавь чек-лист в формате:
### ✅ Чек-лист: что запомнить из главы
- [ ] Пункт 1
- [ ] Пункт 2
```

### 2.2. Написание с учётом шаблона формата
```
Напиши [ГЛАВУ/РАЗДЕЛ], строго соблюдая «Шаблон формата текста». Обрати особое внимание на:
- Выделение терминов **жирным** при первом упоминании
- Длину абзацев (3-5 предложений)
- Чек-лист в конце
- Текстовые дубли, если будут таблицы

Содержание: [ОПИСАНИЕ]
```

### 2.3. Написание введения
```
Напиши введение к книге «[НАЗВАНИЕ]». Оно должно:
1. Захватить внимание читателя (начать с боли/проблемы)
2. Объяснить, почему эта тема важна
3. Кратко представить структуру книги
4. Дать читателю понять, что он получит в итоге

Объём: 2-3 страницы.
Тон: [ТОН].
Обращение: на «[ты/вы]».
```

### 2.4. Написание заключения
```
Напиши заключение к книге «[НАЗВАНИЕ]». Оно должно:
1. Подвести итог всему сказанному
2. Напомнить читателю главные выводы
3. Вдохновить на действия
4. Оставить послевкусие

Объём: 1-2 страницы.
Тон: [ТОН].
Обращение: на «[ты/вы]».
```

## 3. ЗАПРОСЫ ДЛЯ РЕДАКТУРЫ

### 3.1. Редактура готового текста
```
# ЗАПРОС НА РЕДАКТУРУ

**Роль:** Редактор-стилист

**Задача:** Отредактировать приложенный текст.

**Требования:**
- Исправить стилистические ошибки
- Улучшить читаемость
- Сократить «воду»
- Проверить терминологию
- Сохранить авторский голос

**Формат:** Показать исходный и исправленный вариант (можно в виде diff).

**Ограничения:**
- Не менять смысл
- Не добавлять новую информацию
- Сохранить структуру абзацев
```

### 3.2. Проверка научной достоверности
```
# ЗАПРОС НА ПРОВЕРКУ ФАКТОВ

**Роль:** Научный редактор

**Задача:** Проверить приложенный текст на научную достоверность.

**Найди и отметь:**
1. Фактические ошибки
2. Устаревшие данные
3. Спорные утверждения, требующие ссылок
4. Термины, которые нужно объяснить лучше

Для каждого замечания предложи:
- Вариант исправления
- Ссылку на источник (если есть)

**Важно:** Раздели замечания на критические (меняют смысл) и рекомендательные.
```

### 3.3. Сокращение текста
```
Сократи приложенный текст до [N] страниц / [N]% от оригинала.

**Требования:**
- Сохранить все ключевые мысли
- Оставить примеры (самые яркие)
- Убрать повторы
- Упростить сложные конструкции
- Сохранить авторский голос

Покажи сокращённый вариант и отметь, что убрал.
```

### 3.4. Адаптация под аудиоформат
```
Адаптируй приложенный текст для аудиоформата.

**Требования:**
- Убрать отсылки к таблицам или заменить их текстовыми дублями
- Упростить сложные предложения
- Добавить словесные связки
- Убрать визуальные элементы (скобки, списки, если мешают)
- Сделать текст плавным для восприятия на слух

**Формат:** Готовый текст для озвучки.
```

## 4. ЗАПРОСЫ ДЛЯ СОЗДАНИЯ ВСПОМОГАТЕЛЬНЫХ МАТЕРИАЛОВ

### 4.1. Создание глоссария
```
На основе готовой книги «[НАЗВАНИЕ]» составь глоссарий ключевых терминов.

**Формат:** Таблица или список:
- **Термин** — краткое определение (1-2 предложения)

**Требования:**
- Только термины, важные для понимания
- Определения простым языком
- Алфавитный порядок

**Объём:** [N] терминов.
```

### 4.2. Создание аннотации
```
Напиши аннотацию (описание) для книги «[НАЗВАНИЕ]».

**Аннотация должна:**
- Заинтриговать потенциального читателя
- Объяснить, о чём книга
- Показать, кому будет полезна
- Содержать ключевое сообщение

**Варианты:**
1. Короткая (1 абзац) — для магазинов
2. Развёрнутая (3-4 абзаца) — для сайта автора

**Тон:** [ТОН].
```

### 4.3. Создание оглавления с аннотациями глав
```
Составь развёрнутое оглавление книги «[НАЗВАНИЕ]» с краткой аннотацией (2-3 предложения) для каждой главы.

**Формат:**
## Раздел I. [НАЗВАНИЕ]
### Глава 1. [НАЗВАНИЕ]
[Аннотация главы]
### Глава 2. [НАЗВАНИЕ]
[Аннотация главы]
...

**Цель:** Читатель, просматривая оглавление, сразу понимает, о чём каждая глава и стоит ли её читать.
```

### 4.4. Создание чек-листов
```
На основе главы «[НАЗВАНИЕ]» составь подробный чек-лист для читателя.

**Формат:**
### ✅ Чек-лист: [название]
- [ ] Пункт 1 (конкретное действие или вопрос для самопроверки)
- [ ] Пункт 2
- [ ] Пункт 3
...

**Требования:**
- Каждый пункт — практический и проверяемый
- Язык простой и понятный
- Никаких общих фраз, только конкретика
```

## 5. ЗАПРОСЫ ДЛЯ РАБОТЫ С КОНТЕКСТОМ

### 5.1. Начало сессии (после перерыва)
```
Мы работаем над книгой «[НАЗВАНИЕ]». Вот наш Мастер-файл контекста:

[ВСТАВИТЬ ТЕКСТ МАСТЕР-ФАЙЛА]

Пожалуйста, ознакомься и подтверди, что готов продолжить с того места, где мы остановились. Напомни мне, что было сделано в прошлый раз и какой следующий шаг.
```

### 5.2. Продолжение с того же места
```
Продолжай. Вот последний кусок текста, который мы написали:

[ТЕКСТ]

Напиши следующие [N] страниц / закончи мысль.
```

## 6. ЭКСТРЕННЫЕ ЗАПРОСЫ

### 6.1. Когда застрял
```
Я застрял на [ГЛАВА/МЕСТО]. Не могу двинуться дальше. Вот что у меня есть:
[ТЕКСТ]

Задай мне 5 вопросов, которые помогут найти выход. Не предлагай готовых решений — только вопросы, которые заставят меня самого увидеть путь.
```

### 6.2. Когда нужна мотивация
```
Я выгорел. Напомни мне:
- Зачем я это пишу?
- Кто меня ждёт?
- Что изменится, если книга появится?
- Почему это важно именно сейчас?

Напиши мне письмо от имени читателя, который ещё не знает, что его спасёт моя книга.
```

### 6.3. Когда нужна свежая идея
```
Мне нужно [N] свежих идей для [ТЕМА]. Предложи [N] неожиданных подходов, ракурсов или метафор. Можно смелые, даже провокационные. Мы потом отсеем.
```

## 7. ЗАПРОСЫ ДЛЯ ПРОВЕРКИ

### 7.1. Стресс-тест книги
```
# ЗАПРОС НА СТРЕСС-ТЕСТ

**Роль:** Самый строгий критик

**Задача:** Найди 10 слабых мест в книге / главе / концепции:

1. Логические дыры
2. Слабые аргументы
3. Места, где читатель заскучает
4. Противоречия
5. Банальности
6. Недосказанность (где нужно сказать яснее)
7. Перегруженность
8. Фальшивые эмоции
9. Потеря фокуса
10. Слабый финал

Для каждого слабого места предложи 2 способа исправления.
```

### 7.2. Тест на уникальность
```
Чем моя книга отличается от других книг на эту тему? Проанализируй рынок и скажи:
- Что в ней уникального?
- Что могут критиковать?
- Как усилить уникальность?
- Какие конкуренты самые опасные и почему?
```

## 8. ТЕХНИЧЕСКИЕ ЗАПРОСЫ

### 8.1. Конвертация в Markdown
```
Приведи приложенный текст в формат Markdown с правильной иерархией заголовков:
- # — только название книги
- ## — разделы
- ### — главы
- #### — подглавы

Также оформи все списки и выделения.
```

### 8.2. Создание таблицы в Markdown
```
Создай таблицу в Markdown по данным:

[ДАННЫЕ]

После таблицы добавь текстовый дубль для аудиоформата.
```

## 9. ЗАПРОСЫ ДЛЯ ФИНАЛА

### 9.1. Написание благодарностей
```
Напиши раздел «Благодарности» для книги «[НАЗВАНИЕ]». Поблагодари:
- [СПИСОК КОГО]

Стиль: тёплый, искренний, без пафоса. Объём: 1 страница.
```

### 9.2. Написание посвящения
```
Напиши 3 варианта посвящения для книги:
1. Короткое (одна строка)
2. Тёплое (развёрнутое)
3. Метафорическое (через образ)

Книга посвящается: [КОМУ], потому что [ПОЧЕМУ].
```

### 9.3. Ритуал завершения
```
Опиши ритуал завершения книги красиво, как сцену из фильма. Что я чувствую? Что вижу? Кто рядом? Что звучит? Что происходит с книгой в этот момент?

Это описание станет моей мотивацией на всё время написания.
```

## 10. ШАБЛОН ДЛЯ СОЗДАНИЯ НОВОГО ЗАПРОСА

Если ни один из готовых запросов не подходит, используй эту форму:

```
# ЗАПРОС

**Роль:** [кем я должен быть]

**Контекст:** [где мы находимся, что уже сделано]

**Задача:** [что именно нужно сделать]

**Исходный материал:** [что даю]

**Формат вывода:** [как должен выглядеть результат]

**Ограничения:**
- Объём: [сколько]
- Тон: [какой]
- Запреты: [чего нельзя]
- Приоритеты: [на что обратить внимание]
- Дополнительно: [что ещё важно]
```
