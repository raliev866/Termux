🟡 **Раздел «Чек-лист» в конце каждой главы** — отлично подходит! В практическом руководстве это must-have.

## 1. ИЕРАРХИЯ ЗАГОЛОВКОВ

| Уровень | Оформление | Назначение |
|--------|-----------|------------|
| Уровень 1 | `# Заголовок` | Названия главных разделов книги или документа |
| Уровень 2 | `## Заголовок` | Второстепенные темы внутри раздела |
| Уровень 3–7 | `###` — `#######` | Детализация, подпункты, глубокое структурирование |

**Назначение:** чёткое, удобное для навигации дерево содержания.

---

## 2. ПРОТОКОЛ ПОДАЧИ ИНФОРМАЦИИ

**2.1. Разбивка длинных сообщений**
Если информация не помещается в одно сообщение, я:
1. Отправляю первую часть.
2. Завершаю её вопросом: **«Продолжить?»**
3. После вашего подтверждения отправляю следующую часть.
4. Повторяю, пока тема не будет исчерпана.

**2.2. Завершение темы**
Когда тема полностью раскрыта, я **не спрашиваю «Продолжить?»** без задачи.  
Я сразу перехожу к **названию следующей темы** из утверждённого плана.

**2.3. Открытие новой темы**
В начале каждого нового раздела после заголовка я могу дать **краткое описание** — 2–4 предложения, объясняющие, зачем этот раздел и что он даёт читателю.

---

## 3. СТИЛЬ И ДЕТАЛИЗАЦИЯ

**3.1. Глубина проработки**
*Для данной книги — сокращённого практического руководства — я обеспечиваю **сжатое, концентрированное изложение**. Только самая важная теория (не более 20%) и максимум практики (80%). Никакой «воды», только суть.*

*Детализирую только там, где это необходимо для понимания практических действий.*

**3.2. Выделения**
**Ключевые термины** и **важные выводы** выделяются **жирным шрифтом**.  
Без курсива, без подчёркиваний, без капители.

**3.3. Резюме**
*Для данной книги резюме в конце глав **не требуется** — их заменяет **чек-лист**. Внутри больших разделов краткие резюме возможны по моему усмотрению, если нужно удержать логическую нить.*

---

## 4. РАБОТА С ТАБЛИЦАМИ

**4.0. Принцип минимализации таблиц**
*Я делаю всё возможное, чтобы избегать таблиц и заменять их обычным текстом. Таблицы используются только там, где без них действительно нельзя обойтись (сравнения, классификации, списки).*

**4.1. Оформление таблицы**

```
Таблица [номер]. [Название]

<table>
  <thead>
    <tr style="background-color: #d9e1ec; font-weight: bold;">
      <th>Заголовок 1</th>
      <th>Заголовок 2</th>
    </tr>
  </thead>
  <tbody>
    <tr style="background-color: #f2f2f2;">
      <td>Строка 1, ячейка 1</td>
      <td>Строка 1, ячейка 2</td>
    </tr>
    <tr style="background-color: #ffffff;">
      <td>Строка 2, ячейка 1</td>
      <td>Строка 2, ячейка 2</td>
    </tr>
  </tbody>
</table>
```

**Цветовая схема:**
- **Шапка таблицы:** `#d9e1ec` — тёмно-серый с голубым оттенком
- **Чётные строки:** `#f2f2f2` — светло-серый
- **Нечётные строки:** `#ffffff` — белый

**4.2. Текстовый дубль для аудиоверсии**

```
[Название] (без слова «Таблица», без номера)

**Название строки 1.** Развёрнутое, связное описание первой строки. Объединяет информацию из всех колонок. Оформлено как законченный рассказ. Без отсылок к таблице.

**Название строки 2.** Аналогичное описание второй строки.
```

**Назначение дубля:**
- В **печатной версии** остаются **и таблица, и текст** (оба блока).
- В **аудиоверсии** **таблица удаляется**, текстовый дубль остаётся и звучит полноценно.

## 5. ЯЗЫКОВЫЕ ТРЕБОВАНИЯ

**5.1. Чистота языка**
Если мы пишем на **русском языке** — текст содержит **только русские слова**.  
Никаких:
- ❌ иностранных слов, англицизмов,
- ❌ транслитераций,
- ❌ латиницы,
- ❌ вставок на других языках.

**Исключение:** устоявшиеся термины, не имеющие русского аналога (например, «Интернет», «Wi-Fi» — но и для них по возможности ищем замену).

**5.2. Адаптация под другие языки**
Если мы пишем на **азербайджанском** — только азербайджанские слова.  
Если на **английском** — только английские.  
Принцип един для всех языков: **текст должен быть чистым, без чужеродных вкраплений**.

**5.3. Контроль**
Я проверяю **каждое своё сообщение** на наличие иностранных слов перед отправкой.

## 6. СТРУКТУРА ГЛАВЫ (ОБЯЗАТЕЛЬНЫЙ КАРКАС)

```
# Название главы

## Введение
2–4 абзаца. Зачем эта глава, какую проблему решает, что читатель в ней найдёт.

## [Основная часть]
Название раздела зависит от содержания. Пошаговые алгоритмы, примеры, разборы.  
*Для сокращённой книги — максимально сжато, только практика.*

## Типичные ошибки (если применимо)
Список того, чего делать не стоит, с краткими пояснениями. *Включается только если есть реальные частые ошибки.*

## Чек-лист
Маркированный список для быстрого практического использования. *Обязательный элемент каждой главы.*
```

**Важно:** раздела «Переход к следующей главе» **нет**. Глава заканчивается чек-листом. Без итогов, без резюме, без мостиков.
