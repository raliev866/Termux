




Чтобы Termux «увидел» твою команду `генератор` (которая лежит в папке `bin`), нужно выполнить два простых шага: убедиться, что у файла есть права на запуск, и добавить путь к этой папке в переменную среды `PATH`.

### Шаг 1: Сделай файл исполняемым

Даже если файл лежит в `bin`, Termux не даст его запустить, пока у него нет прав на исполнение. Выполни:

```bash
chmod +x /storage/emulated/0/Documents/15_Настройки_скрипты_Termux_бекап_рестор/Алиасы_команды_SH/bin/генератор

```

### Шаг 2: Добавь папку в системный путь (PATH)

Чтобы не прописывать каждый файл отдельно, мы добавим твою папку `bin` в список мест, где Termux ищет команды.

Выполни эту команду один раз (она допишет путь в твой `.bashrc`):

```bash
echo 'export PATH="$PATH:/storage/emulated/0/Documents/15_Настройки_скрипты_Termux_бекап_рестор/Алиасы_команды_SH/bin"' >> ~/.bashrc
source ~/.bashrc

```

### Шаг 3: Проверка

Теперь просто введи в терминале:
`генератор`

---

### Почему это лучше, чем алиас?

Когда ты добавляешь папку в `$PATH`, тебе **не нужно** создавать отдельные алиасы для каждого нового скрипта. Как только ты положишь в эту папку `bin/` любой исполняемый файл (без расширения), Termux сразу будет узнавать его как команду.

**Если команда выдаст ошибку "Permission denied" или "Command not found":**

1. Введи `ls -l /storage/emulated/0/Documents/15_Настройки_скрипты_Termux_бекап_рестор/Алиасы_команды_SH/bin/` — давай проверим, есть ли там файл `генератор` и видит ли его система.
2. Скопируй сюда ошибку, если она появится.

Чтобы Termux начал воспринимать файл `генератор` как полноценную команду, нужно решить две задачи: дать ему **права на запуск** (сейчас это просто текстовый файл `-rw-rw----`) и сообщить системе, где его искать.

### 1. Выдай права на исполнение

В Android/Termux файлы на внутренней памяти (`/storage/emulated/0/...`) часто защищены от прямого исполнения. Поэтому мы сначала скопируем файл в системную папку Termux, где он точно сможет работать.

Выполни эти команды по очереди:

```bash
# Создаем локальную папку bin в самом Termux (если её нет)
mkdir -p $HOME/bin

# Копируем твой генератор туда
cp /storage/emulated/0/Documents/15_Настройки_скрипты_Termux_бекап_рестор/Алиасы_команды_SH/bin/генератор $HOME/bin/

# Делаем его исполняемым
chmod +x $HOME/bin/генератор

```

### 2. "Познакомь" Termux с этой папкой

Теперь нужно сказать Termux: «Смотри в папку `$HOME/bin`, когда я ввожу команды». Для этого добавим путь в файл автозагрузки `.bashrc`.

```bash
# Добавляем путь в конфиг
echo 'export PATH="$HOME/bin:$PATH"' >> ~/.bashrc

# Обновляем текущую сессию
source ~/.bashrc

```

---

### 3. Проверь результат

Теперь просто введи:

```bash
генератор

```

Если скрипт запустился — поздравляю, ты только что создал свою собственную системную команду!

### Почему мы копировали файл в `$HOME/bin`?

Файловая система Android (на которой лежит твоя папка `Documents`) не поддерживает флаг «исполняемый» (`+x`) из соображений безопасности. Чтобы скрипт работал как команда, он должен находиться во внутреннем хранилище Termux (в папке `$HOME`).
