# ПОЛНАЯ ИНСТРУКЦИЯ: КАК ПРОВОДИТСЯ РЕОРГАНИЗАЦИЯ ПО КРИТИКЕ

## Введение в процесс

Реорганизация книги (или любого большого текста) — это как **ремонт в квартире**. Сначала вы оцениваете, что работает, а что нет (критика), затем создаёте новый план (архитектура), и только потом начинаете физически перемещать вещи (переименование/перемещение файлов).

---

## ЭТАП 1: ПОДГОТОВКА И АНАЛИЗ

### 1.1. Сбор данных
Перед любой критикой нужно увидеть полную картину:

```
# Команда для получения полной структуры
find . -type f -name "*.md" | sort > struktura.txt
```

Или используйте красивый вывод:
```
tree -L 4 > derevo.txt
```

### 1.2. Изучение оглавления
Распечатайте или откройте оглавление рядом со структурой папок. Спросите себя:
- **Логична ли последовательность?** (От простого к сложному? От теории к практике?)
- **Сбалансированы ли разделы?** (Нет ли томов с 20 главами и других с 3?)
- **Есть ли "висящие" темы?** (Темы, которые не вписываются в соседние разделы)

### 1.3. Поиск проблемных зон
Отметьте маркерами:
- 🔴 **Критические проблемы** (ломают понимание)
- ⚠️ **Серьёзные недочеты** (мешают чтению)
- ✅ **Сильные стороны** (оставляем как есть)

---

## ЭТАП 2: КРИТИКА (РОЛЬ "АРХИТЕКТОР")

### 2.1. Что проверяет архитектор

| Аспект | Что ищем | Пример вопроса |
|--------|----------|----------------|
| **Структура** | Иерархия, вложенность | "Не слишком ли глубоко зарыта эта глава?" |
| **Баланс** | Равномерность разделов | "Почему Том 4 в 3 раза больше Тома 2?" |
| **Логика** | Последовательность | "Можно ли понять главу 7 без главы 3?" |
| **Полнота** | Охват тем | "Все ли обещанные в оглавлении темы раскрыты?" |
| **Навигация** | Удобство для читателя | "Как новичку найти нужную информацию?" |

### 2.2. Формат критики
Хорошая критика должна содержать:

1. **Что именно не так** (конкретное место)
2. **Почему это проблема** (последствия для читателя)
3. **Как исправить** (конкретное предложение)

**Плохо:** "Том 4 плохой"
**Хорошо:** "В Томе 4 11 глав подряд без внутренней структуры. Читатель теряется. Предлагаю разбить на 4 подраздела по системам организма."

## ЭТАП 3: СОЗДАНИЕ НОВОГО ПЛАНА

### 3.1. Новая структура
На основе критики создаётся идеальная структура:

```
Том 1. Название
├── Раздел 1. Название
│   ├── Глава 1. Название
│   └── Глава 2. Название
└── Раздел 2. Название
    └── Глава 3. Название
```

### 3.2. Маппинг (связь старого с новым)
Создаётся таблица соответствия:

| Старый путь | Новый путь | Действие |
|-------------|------------|----------|
| Том 4/Глава 34.md | Том 4/Подраздел А/Глава 34.md | Переместить |
| Том 5/Раздел XIII | Том 5/Раздел XI | Переименовать и переместить |
| - | Том 4/Введение.md | Создать новый |

### 3.3. Проверка связей
Убедитесь, что после перемещения не нарушатся внутренние ссылки. Если в тексте есть "как мы обсуждали в главе 12", после перемещения это нужно исправить.

## ЭТАП 4: ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ (TERMUX)

### 4.1. Команды для работы с файлами

**Переименование папки:**
```bash
mv "Старое название" "Новое название"
```

**Перемещение файла:**
```bash
mv "старый/путь/файл.md" "новый/путь/файл.md"
```

**Создание новой папки:**
```bash
mkdir -p "Том 4/Подраздел А"
```

**Массовое перемещение:**
```bash
# Переместить все главы 34-44 в новый подраздел
mv "Том 4/Глава "{34..44}".md" "Том 4/Подраздел А/" 2>/dev/null
```

### 4.2. Безопасность (ОЧЕНЬ ВАЖНО!)

**Правило №1: Никогда не работайте без резервной копии!**

```bash
# Создаём резервную копию всей структуры
cd /storage/emulated/0/Documents/
cp -r "Кислотно-щелочной баланс" "Кислотно-щелочной баланс_BACKUP_$(date +%Y%m%d)"
```

**Правило №2: Проверяйте каждый шаг**

```bash
# Перед выполнением команды посмотрите, что будет перемещено
ls "Том 4/Глава "{34..44}".md" 2>/dev/null
```

**Правило №3: Используйте `-v` (verbose) для отслеживания**

```bash
mv -v "старый" "новый"  # Покажет, что куда перемещается
```

### 4.3. Пошаговое выполнение

**Лучше делать маленькими шагами и проверять:**

```bash
# ШАГ 1: Создаём новую структуру
mkdir -p "Том 4/Подраздел А. Опорно-двигательная система"
mkdir -p "Том 4/Подраздел Б. Сердечно-сосудистая и нервная"
# ... и так далее

# ШАГ 2: Перемещаем главы
mv -v "Том 4/Глава 34.md" "Том 4/Подраздел А/"
mv -v "Том 4/Глава 35.md" "Том 4/Подраздел А/"

# ШАГ 3: Проверяем результат
ls -la "Том 4/Подраздел А/"

# ШАГ 4: Переходим к следующему шагу
```

## ЭТАП 5: ПРОВЕРКА И ТЕСТИРОВАНИЕ

### 5.1. Визуальная проверка
```bash
# Посмотрите новую структуру
tree -L 4 "Том 4"
```

### 5.2. Проверка целостности
Убедитесь, что все файлы на месте:
```bash
# Было 44 главы в Томе 4? Проверим, сколько стало
find "Том 4" -name "Глава *.md" | wc -l
```

### 5.3. Проверка внутренних ссылок (если есть)
Если в тексте есть ссылки на другие главы, их нужно будет исправить позже на этапе редактуры.

## ЭТАП 6: ДОКУМЕНТИРОВАНИЕ ИЗМЕНЕНИЙ

### 6.1. Создайте лог изменений
Создайте файл `CHANGELOG.md` в корне проекта:

```markdown
# Журнал реорганизации

## 2026-03-12
- Том 4 разбит на 4 подраздела
- Главы 34-44 перемещены в соответствующие подразделы
- Разделы XI и XIII в Томе 5 поменяны местами
- Добавлено введение к Тому 1
```

### 6.2. Обновите оглавление
После реорганизации нужно создать новое оглавление, отражающее изменения.

## ТИПИЧНЫЕ ОШИБКИ НОВИЧКОВ (И КАК ИХ ИЗБЕЖАТЬ)

| Ошибка | Последствия | Как избежать |
|--------|-------------|--------------|
| Работа без бэкапа | Потеря данных | Всегда копируйте перед началом |
| Слишком длинные команды | Трудно отлаживать | Делайте по одному шагу |
| Игнорирование пробелов в именах | Команды не работают | Используйте кавычки: "Имя с пробелами" |
| Перемещение вместо копирования | Нельзя откатить | Сначала копируйте, потом удаляйте |
| Нет проверки промежуточных результатов | Ошибка накапливается | Проверяйте после каждого шага |


### Шаг 2: Критика
```
"11 глав в одном разделе без подразделов - это перегружает читателя"
```

### Шаг 3: План
```
Том 4
├── Подраздел А. Опорно-двигательная система (главы 34-35)
├── Подраздел Б. Сердечно-сосудистая и нервная (36-37)

└── Подраздел Г. Системные заболевания (41-44)
```

### Шаг 4: Реализация в Termux
```bash
# Бэкап
cp -r "Том 4" "Том 4_BACKUP"

# Создание подразделов
mkdir -p "Том 4/Подраздел А. Опорно-двигательная система"
mkdir -p "Том 4/Подраздел Б. Сердечно-сосудистая и нервная"
mkdir -p "Том 4/Подраздел В. Висцеральные системы"
mkdir -p "Том 4/Подраздел Г. Системные заболевания"

# Перемещение
mv -v "Том 4/Глава 34.md" "Том 4/Подраздел А/"
mv -v "Том 4/Глава 35.md" "Том 4/Подраздел А/"
mv -v "Том 4/Глава 36.md" "Том 4/Подраздел Б/"
mv -v "Том 4/Глава 37.md" "Том 4/Подраздел Б/"
# ... и так далее
```

### Шаг 5: Проверка
```bash

```
